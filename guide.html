Setup Guide

01
Run the setup script

Open a terminal inside the ghostcrawler/ folder. The script builds the MCP server and writes the VS Code config automatically — no manual editing needed.

Mac/Linux ./setup.sh
Windows powershell -ExecutionPolicy Bypass -File setup.ps1

Requires Node.js ≥ 18 — nodejs.org

02
Load the Chrome extension

Go to chrome://extensions and enable Developer Mode (top-right toggle). Click Load Unpacked and select the ghostcrawler/ folder. Pin the icon to your toolbar.

03
Set up Burp Suite

In Burp Suite go to Extensions → BApp Store, search MCP, and install Burp MCP. It will listen on 127.0.0.1:9876.

Set your browser proxy to 127.0.0.1:8080 so all traffic routes through Burp.

04
Restart VS Code

Fully quit and reopen VS Code. GitHub Copilot will auto-connect to the GhostCrawler MCP server on startup.

👁
Choose your Attack Mode

In the extension popup, toggle the Attack mode button to control how exploits are fired.

Silent DEFAULT

All attacks fire through the Burp proxy in the background. Browser stays idle. Fastest — recommended for large scans.

Live 👁

Browser navigates to each attack URL so you can watch exploits fire in real time. Great for demos and walkthroughs.

05
Activate & Hunt

Navigate to your target in Chrome. Open GitHub Copilot Chat in VS Code and run:

Copilot /ghostcrawler pentest_active_tab

The HUD will appear on the target page. All findings auto-log to Burp Repeater.

Command Reference

Command & Syntax What it does
/ghostcrawler pentest_active_tab Full OWASP Top 10 scan — source review, fingerprint, crawl, XSS/SQLi/auth. Default entry point. No arguments needed.
/ghostcrawler gc_doctor Diagnose all connections: bridge, extension, Burp MCP. Run this first on any failure.
/ghostcrawler auto_crawl
e.g. auto_crawl depth 2, attacks xss sqli		 Crawl and attack all forms and buttons. Optionally specify maxDepth and which attacks to run.
/ghostcrawler get_attack_surface Map all forms, buttons, and API endpoints on the active tab before attacking.
/ghostcrawler run_browser_plan
e.g. navigate /login, type admin in #user, click submit		 Execute precise browser steps: navigate, type, click, extract_text. Use for custom exploit chains.
/ghostcrawler run_attack_command
e.g. xss on input[name='q'] with payload <img src=x onerror=alert(1)>		 Fire one attack type: xss, manual-sqli, default-creds, http-verb, custom-request. Specify target selector and payload.
/ghostcrawler observe_dom_changes
e.g. watch #comments for 5000ms		 Watch live DOM mutations. Optionally scope to a CSS selector and set durationMs. Use after XSS to confirm payload fired.
/ghostcrawler check_burp_mcp Verify Burp Suite MCP connection on port 9876 and list available tools.

Troubleshooting

!
MCP server not reachable

The extension reports Bridge ✗ or VS Code cannot connect to GhostCrawler.

Check if the server is already running, then restart it:

Terminal lsof -ti:3200 | xargs kill -9 2>/dev/null; sleep 1; node /path/to/ghostcrawler/mcp-server/dist/index.js

Multiple processes can hold port 3200 — the lsof | xargs kill clears them first.

!
"Receiving end does not exist" / Extension error

The content script isn't injected on the current page.

  1. Navigate to chrome://extensions
  2. Find GhostCrawler and click the reload ↺ icon
  3. Go back to your target tab and refresh it
  4. Run /ghostcrawler gc_doctor to verify
!
Findings not appearing in Burp

Burp Suite MCP is not connected or the proxy isn't set up.

  1. Ensure Burp Suite is open with the MCP Server extension installed and listening on port 9876
  2. Confirm Chrome is proxying through 127.0.0.1:8080
  3. Run /ghostcrawler check_burp_mcp in Copilot Chat
!
HUD shows "Complete" but AI is still working

This was a known bug — fixed in the current version. If it reappears:

  1. Rebuild the server: cd mcp-server && npm run build
  2. Restart the server (see command above)
  3. Reload the extension at chrome://extensions
!
"Scan stopped by user" — but you didn't stop it

The extension heartbeat was lost during a slow page navigation. Fixed in current version (heartbeat now in background script). If it persists, reload the extension and restart the server.

!
Browser not moving / AI attacking via curl only

Passive checks (headers, CORS, cookies, etc.) always run server-side — that's by design. For active attacks, toggle Attack Mode → Live 👁 in the extension popup to make the browser navigate to each attack URL.

Silent mode is faster and less noisy — recommended for large scans. Live mode is better for demos.
